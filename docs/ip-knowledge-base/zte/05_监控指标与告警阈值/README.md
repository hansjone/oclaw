# 05_监控指标与告警阈值

## 定位
定义各协议的运维监控标准和告警阈值，解决"什么算异常"的问题。这是实现主动运维和故障预测的基础。

## 内容要求
- **路由表规模阈值**：BGP/OSPF/IS-IS的路由表项上限告警值
- **会话状态指标**：邻居震荡频率、Hold Timer超时次数等
- **资源利用率**：CPU/内存/LCAM使用率的告警门限
- **流量特征基线**：正常时段的流量范围，偏离基线时告警

## RAG作用
当用户问"BGP路由表多少条算异常"或"LDP邻居多久震荡一次需要关注"时，AI索引这里的量化阈值标准。

## 为什么需要量化阈值？
- **避免误报**：没有基线的告警只是噪音
- **提前预警**：在故障发生前发现趋势性异常
- **标准化运维**：不同值班人员对"异常"有统一判断标准
